# Security Policy

## Reporting a vulnerability

If you find a security problem in AFM Studio, please report it privately. Do not open a public issue for security reports.

Email taylor@techopolis.app or mikedoise@icloud.com with:

- A description of the problem.
- Steps to reproduce it.
- The impact you think it could have.
- Any logs or proof of concept, with secrets removed.

You can expect an acknowledgement within a few days. We will work with you on a fix and let you know when it ships. Please give us a reasonable window to address the issue before sharing it publicly.

## Supported versions

AFM Studio tracks the OS 27 and Xcode 27 beta cycle, so the latest commit on the default branch is the supported version. Fixes land there first.

## Handling secrets

Never post secrets, API keys, provisioning profiles, signing identities, private model URLs, or proprietary model artifacts in issues, pull requests, or logs. If a credential is exposed, rotate it and let us know.
